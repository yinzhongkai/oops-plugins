# 公众号适配规范

adapt 适配公众号（WeChat Official Account）的完整规范。本文件定义标题、结构、金句、排版。

## 1. 平台调性

公众号是长文为主的图文平台，读者在碎片时间深度阅读。调性：**有信息增量的深度表达**，比知乎更生活化、比小红书更完整。可以是知识科普、观点论述、个人成长、行业观察。篇幅通常 1500-3500 字。

## 2. 标题钩子

标题决定打开率。基于 source.md 核心观点拟 3-5 个候选，挑最契合的。常见钩子类型：

- **反常识**：挑战默认认知。如「为什么越努力的人越容易陷入穷忙」
- **具体数字/承诺**：给出明确预期。如「读完这章我才明白的 3 个投资误区」
- **痛点直击**：点中读者困境。如「总是存不下钱？问题可能不在收入」
- **悬念/故事**：用故事或悬念勾起好奇。如「巴菲特说市场先生那天，我愣住了」
- **身份共鸣**：唤起目标受众身份认同。如「30 岁还在迷茫的人，都该读读这一章」

要求：标题含核心观点的影子但不剧透结论；避免标题党空洞承诺；不超过 30 字为宜。最终标题由用户从候选中定。

## 3. 正文结构

公众号长文要有清晰的阅读路径。推荐结构（按 source.md 素材灵活调整）：

```
开头（钩子 + 共鸣）
  → 用一个场景/故事/反问把读者拉进来，点出痛点或悬念
  → 引出本文要讲的核心观点（呼应标题）

主体（2-4 个分论点）
  → 每个分论点：观点句 + 论据/案例 + 读者能共鸣的解释
  → 分论点之间有过渡，层层递进而非平行罗列
  → 用 source.md 的关键论据素材填充

结尾（升华 + 价值）
  → 回扣核心观点，升华到更大的意义或行动建议
  → 可用 source.md 的可引用金句收尾
  → 避免硬塞「关注点赞在看」式的机械结尾，自然收束
```

分论点不宜超过 4 个，否则读者疲劳。每个分论点一个清晰的小标题。

## 4. 金句点睛

公众号靠金句被转发。把 source.md 的可引用金句用在关键位置：

- **开头**：一句精炼观点开场，定调。
- **分论点首尾**：每个分论点用一个金句锚定记忆。
- **结尾**：用一个金句收尾，提升传播力。

金句要短、要有画面或反差、要能脱离上下文独立成立。source.md 金句不够时，可在 source.md 已有理解基础上**提炼**（不是编造新观点），保持原创性。

## 5. 排版

公众号阅读在手机端，排版影响完读率：

- **短段落**：每段 2-4 行，手机屏一屏可见多个段落，降低压迫感。
- **留白**：段落间空行，关键句独立成段强化视觉。
- **小标题**：分论点用小标题分隔，便于扫读。
- **重点标注**：核心金句可加粗或用引用块，但克制——满篇加粗等于没加粗。
- **少用复杂格式**：表格、代码块在公众号体验差，优先纯文字 + 加粗 + 引用。

## 6. 语感

公众号语感比知乎口语、比小红书正式：

- 像在和一个聪明的朋友聊天，有温度但不轻浮。
- 可以有个人口吻（「我读到这儿时…」「这章让我想起…」），保留 source.md 里读者自己的表达习惯。
- 避免 AI 腔：不用「首先其次最后」机械排比、不用「在当今社会」式空洞开场、不过度对仗。
- 避免说教腔：不下指令、不居高临下，用「一起来看」「也许」等平等表达。

## 7. 从 source.md 取材

| source.md 字段 | 公众号用途 |
|----------------|-----------|
| 核心观点 | 标题影子 + 结尾升华的主轴 |
| 关键论据素材 | 主体分论点的论据/案例 |
| 可引用金句 | 开头定调、分论点锚定、结尾收尾 |
| 受众 | 决定开场共鸣点与举例贴近度 |
| 备注 | 守禁忌；「待深入」点不强行展开 |

## 8. 自检

产出前自检：

- [ ] 标题含核心观点影子、不剧透、不标题党？
- [ ] 开头 3 句内把读者拉进来？
- [ ] 分论点 ≤4 个、层层递进？
- [ ] 金句用在开头/分论点/结尾关键位置？
- [ ] 段落短、有留白、重点克制标注？
- [ ] 无 AI 腔、有个人温度？
- [ ] 未编造 source.md 之外的论据？
- [ ] source.md 备注的禁忌点未触碰？

## 9. 产出

文案写入 `<选题>/wechat.md`，首行可直接是标题（公众号无标题 frontmatter 约定，正文即全部）。
