# 知乎适配规范

adapt 适配知乎的完整规范。本文件定义开头破题、论证层次、专业感、引用。

## 1. 平台调性

知乎是问答/知识社区，读者期待**有逻辑、有信息量、经得起推敲**的回答。调性：理性、专业、有深度，但不要掉书袋。可以是观点论述、经验分享、知识科普、问题分析。篇幅通常 1500-4000 字，重质不重量。

## 2. 开头破题

知乎读者扫得快，开头要迅速立住「这篇值得读」。基于 source.md 核心观点破题，常见方式：

- **直接给结论**：开门见山抛出核心观点，再展开论证。知乎读者吃这套——先给答案再讲为什么。
- **界定问题**：先厘清问题本身（很多人把 X 理解错了），再给正确视角。
- **反常识切入**：抛出与主流不同的观点，勾起探究欲。
- **数据/事实开场**：用一个有力的事实或数据点引出论点。

避免：寒暄废话、长篇故事铺垫（知乎不像公众号吃长故事开场）、空洞的「这个问题很有意思」。

## 3. 论证层次

知乎的灵魂是论证质量。基于 source.md 关键论据素材搭建论证：

```
核心观点（开头已立）
  → 分论点 1：观点句 + 论据（数据/案例/逻辑推演）+ 小结
  → 分论点 2：层层递进，最好能回应潜在反驳
  → ...
  → 综合论证：把分论点收束回核心观点
```

要点：

- **每个分论点有支撑**：不空喊口号，每个论点配论据（source.md 的数据/案例）或清晰逻辑推演。
- **递进而非罗列**：分论点之间有逻辑关系（因果、递进、转折），不是平行堆砌。
- **回应反驳**：预判读者可能的质疑并回应，体现思考完整度——这是知乎高赞回答的共性。
- **承认局限**：source.md 备注的「待深入」点，知乎语境下可坦诚「这点我也还在想」，比硬装懂更显可信。

## 4. 专业感

知乎读者重视专业感，但专业 ≠ 掉书袋：

- **用准术语**：该用专业术语时用，但首次出现配一句通俗解释。
- **有信息增量**：提供读者不知道的事实/视角/数据，而非复述常识。
- **逻辑清晰**：因果、条件、递进关系用连接词标明，让论证链条可追踪。
- **克制断言**：有把握的下断言，没把握的标「倾向于」「可能」，不一概而论。
- **不卖弄**：术语和引述服务于说清问题，不为显得博学而堆砌。

## 5. 引用

知乎重视溯源和可信度：

- **数据/事实引用**：source.md 来自链接/话题的数据，标注来源（「据 X 报道」「X 的研究显示」）。链接来源注意转述，勿照抄。
- **书本引用**：source.md 来自感悟文档（bookworm 读书笔记）时，可引用原书观点，标注书名章节——这恰恰是知乎读者喜欢的「有出处的知识」。
- **金句引用**：source.md 可引用金句用作分论点锚定或结尾，标注是原书原句还是读者自己的话。
- **格式**：引用原句用引用块（>），自己的转述融入正文。

## 6. 语感

知乎语感理性、克制、有思辨味：

- 像一个有阅历的人在认真回答一个问题，不煽情、不抖机灵过度。
- 可以有个人经验（「我读到这一章时的体会是…」），但个人经验服务于论证，不喧宾夺主。
- 避免 AI 腔：不用「首先其次最后」机械排比、不用「综上所述」式八股收尾、不过度对仗。
- 避免说教和绝对化：用「我的理解是」「一种看待方式是」保留思辨空间。

## 7. 结构标记

知乎读者扫读多，结构要清晰：

- 用小标题分隔分论点（如「一、…」「二、…」或语义小标题）。
- 核心观点和关键结论可加粗。
- 长论证用有序列表呈现步骤。
- 但克制——结构标记是辅助阅读，不是炫技。

## 8. 从 source.md 取材

| source.md 字段 | 知乎用途 |
|----------------|---------|
| 核心观点 | 开头直接立论 + 综合论证收束 |
| 关键论据素材 | 分论点的论据/数据/案例，回应反驳 |
| 可引用金句 | 分论点锚定、结尾点睛 |
| 受众 | 决定术语深浅和举例领域 |
| 备注 | 守禁忌；「待深入」点坦诚标注增可信 |

## 9. 自检

产出前自检：

- [ ] 开头迅速破题、无废话铺垫？
- [ ] 每个分论点有论据或逻辑推演支撑？
- [ ] 分论点递进而非平行罗列？
- [ ] 回应了潜在反驳？
- [ ] 引用标注了来源（书本/数据/原句）？
- [ ] 专业但不掉书袋？
- [ ] 无 AI 腔、保留思辨克制？
- [ ] 未编造 source.md 之外的论据？
- [ ] source.md 备注的禁忌点未触碰？

## 10. 产出

文案写入 `<选题>/zhihu.md`。首行可作标题（知乎回答通常无独立标题，以开头破题为入口；若用户要发知乎专栏/文章则需标题，可基于核心观点拟一个）。
