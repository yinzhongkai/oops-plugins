# 链接采集手法

gather 采集网页链接作为素材。本文件定义 WebFetch 抓取、正文清洗、付费墙/反爬/登录墙的报错与兜底。

## 1. 链接采集做什么

用户给一个网址（http/https），要求抓取其内容作为文案素材。用内置 `WebFetch` 工具抓取，抓后清洗正文（去页面框架/广告/导航），只留可用的正文信息留底。

## 2. 抓取

用 WebFetch 抓取目标网址。WebFetch 会把页面转成 markdown，便于提取。

抓取时给的 prompt 应聚焦「提取正文核心信息」，而非泛泛「总结页面」：

- 「提取这篇文章的核心观点、关键论据、可引用的原句金句。」
- 「忽略页面导航、侧边栏、广告、评论区，只提取正文内容。」

## 3. 正文清洗

WebFetch 转出的 markdown 可能混入页面框架。清洗要点：

- **去导航与框架**：页头、页脚、面包屑、侧边栏推荐、相关阅读链接——一律剔除。
- **去广告与推广**：文中插入的推广卡片、赞助内容、下载 App 提示——剔除。
- **去评论区**：读者评论、互动模块——剔除（除非用户明确要参考评论风向）。
- **保留正文**：标题、作者、发布时间、正文段落、正文内的图表说明。
- **保留元信息**：来源平台、作者、发布时间——记入留底 frontmatter，供 organize 判断受众与时效性。

清洗后正文应接近「干净的阅读视图」。

## 4. 提取要点

按 source.md 中间格式字段方向提取：

| source.md 字段 | 从链接正文取 |
|----------------|-------------|
| 核心观点 | 文章主旨、作者核心主张 |
| 关键论据素材 | 支撑观点的论据、数据、案例 |
| 可引用金句 | 文中精炼原句（标注来自原文，adapt 引用时注意版权与转述） |
| 受众 | 从文章主题和语气推断（如职场文 → 职场人群） |
| 备注 | 来源平台、作者、发布时间、是否有时效性 |

注意：链接素材是「别人的内容」。adapt 产出时不得直接抄袭原文当自己的观点——核心观点应是用户基于素材的再表达。在留底备注里标注「此为外部链接素材，adapt 时注意原创性」。

## 5. 付费墙 / 反爬 / 登录墙处理

WebFetch 抓不到完整正文时，按失败原因分流，**一律报错反馈用户，不静默失败**：

| 情况 | 表现 | 处理 |
|------|------|------|
| 付费墙 | 只抓到摘要/导语，正文需付费 | 报错：「该文章有付费墙，只抓到摘要：<摘要>。请手动粘贴正文后重采。」 |
| 反爬 | 抓到验证页/空内容/403 | 报错：「该网站有反爬机制，抓取失败。请手动粘贴正文，或换一个可公开访问的来源。」 |
| 登录墙 | 抓到登录提示页 | 报错：「该内容需登录可见，抓取失败。请手动粘贴正文，或换一个公开来源。」 |
| 链接失效 | 404/超时 | 报错：「链接无法访问（404/超时）。请确认链接正确或换一个来源。」 |

兜底统一为：**让用户手动粘贴正文**。用户粘贴后，按第 3、4 节清洗并提取，等同成功抓取。手动粘贴的正文标注来源类型仍为 `url`，备注里写「手动粘贴」。

## 6. 留底格式

把清洗后正文写入 `<选题>/raw-source.md`：

```markdown
---
来源类型: url
来源链接: <网址>
来源平台: <公众号/知乎/博客/新闻站等>
作者: <作者>
发布时间: <时间，若有>
抓取方式: WebFetch | 手动粘贴
---

# 原始素材留底

## 正文
<清洗后正文>

## 提取要点
### 核心观点
<...>
### 关键论据素材
<...>
### 可引用金句
<...，标注来自原文>

## 备注
外部链接素材，adapt 时注意原创性，勿直接抄袭当自有观点。
```

## 7. 健壮性

- **抓到内容明显是列表页/索引页**（非单篇文章）：报错，请用户给具体文章链接。
- **正文极短**（如只有一两句）：报告素材过薄，提示用户补充或换来源。
- **多链接**：用户一次给多个链接，逐个抓取、各自留底到 `raw-source.md` 分节，选题名由用户定或从主题归纳。
