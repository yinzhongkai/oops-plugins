# 话题采集手法

gather 采集一个话题作为素材。本文件定义 WebSearch 搜资料、参考竞品、提炼搜索要点的做法。

## 1. 话题采集做什么

用户给一个话题描述（自然语言，非文件路径、非网址），要求搜集资料作为文案素材。信息密度最低——话题只是一个方向，需要主动搜集充实。用内置 `WebSearch` 搜网络资料，并参考竞品（看别人怎么写这个话题）。

## 2. 搜索策略

用 WebSearch 围绕话题搜资料。搜索要点：

- **核心词 + 变体**：先用话题核心词搜，再换同义/相关词补搜，覆盖不同表述。
- **聚焦中文结果**：目标平台是中文社媒，优先中文资料。WebSearch 是 US-only，对中文话题覆盖有限——若搜不到足够中文资料，如实告知用户，请其补充背景或提供参考链接。
- **时效**：话题有时效性（如热点事件）时，搜索带时间限定；常青话题不限。

搜 3-5 组关键词即可，不必无限发散。搜太多反而稀释焦点。

## 3. 参考竞品

参考竞品 = 看别人怎么写这个话题，目的是**找角度、找缺口，而非抄内容**：

- **看主流角度**：别人普遍从哪些角度切这个话题？记下常见切入点。
- **找差异化缺口**：哪些角度别人没写透、或与用户立场不同？这是用户文案可发力的差异点。
- **看高赞内容的结构**：竞品高赞内容用了什么钩子、什么结构？供 adapt 参考（但不照搬）。

竞品分析记入留底，organize 整理时可据此建议差异化定位。明确标注：竞品仅供参考角度与结构，adapt 产出须原创，不得抄袭竞品文字。

## 4. 提取要点

话题来源信息最少，提取时注意补全 source.md 字段：

| source.md 字段 | 从话题搜索取 |
|----------------|-------------|
| 核心观点 | 话题本身可能没有明确观点——这常需用户在 organize 阶段定。gather 先记下话题方向和搜到的各方观点。 |
| 关键论据素材 | 搜索到的数据、案例、事实 |
| 可引用金句 | 搜到的精炼表述（标注来源，注意原创性） |
| 受众 | 从话题属性推断（如「考研焦虑」→ 备考学生） |
| 备注 | 各方观点概览、竞品常见角度、差异化建议 |

话题来源最易出现「核心观点缺失」——这正常，因为话题只是方向。gather 如实留底「观点待用户在 organize 阶段确立」，不替用户编观点。

## 5. 留底格式

把搜索要点写入 `<选题>/raw-source.md`：

```markdown
---
来源类型: topic
话题描述: <用户给的话题>
搜索关键词: <用的几组关键词>
---

# 原始素材留底

## 搜到的资料
### 关键事实/数据
<...>
### 各方观点
- 观点A：<...>（来源）
- 观点B：<...>（来源）

## 竞品参考
### 常见切入角度
<...>
### 高赞内容结构
<...>
### 差异化缺口
<别人没写透或与用户立场不同的点>

## 提取要点
### 关键论据素材
<...>
### 可引用金句
<...，标注来源>
### 受众
<...>

## 备注
话题来源，核心观点待用户在 organize 阶段确立。竞品仅供角度参考，adapt 须原创。
```

## 6. 健壮性

- **搜索无结果**：换关键词重试一两次；仍无果则向用户报告，请其细化话题、提供背景、或给参考链接。
- **中文资料不足**：WebSearch 对中文话题覆盖有限，如实告知，请用户补充。
- **话题太宽泛**（如「写职场」）：提示用户收窄到具体切入点，否则 organize 难以确立聚焦的核心观点。
- **话题涉敏感/不宜发布内容**：如实提示风险，由用户决定是否继续。
