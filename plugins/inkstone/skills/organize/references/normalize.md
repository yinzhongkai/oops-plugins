# 中间格式整理规则

organize 把 raw-source.md 归一成 source.md 的详细规则。本文件定义三种来源的整理侧重、字段缺失处理、话题确立观点的引导话术、完整模板。

## 1. source.md 完整模板

```markdown
---
选题: <选题名>
来源类型: book | url | topic
来源路径或链接: <感悟文档路径 / 网址 / 话题描述>
整理时间: <YYYY-MM-DD>
---

## 核心观点
（一句话讲清这篇要表达什么。所有平台文案围绕此主轴展开，不得偏离。）

## 关键论据素材
（支撑观点的论据、数据、例子。adapt 按平台取舍详略。）

## 可引用金句
（适合直接用的原句、感悟、用户原话。尤其小红书钩子、公众号点睛。）

## 受众
（写给谁看：年龄/职业/兴趣/痛点。决定语气和深浅。）

## 备注
（风格倾向、禁忌、平台倾向建议、待深入点、原创性提醒等。）
```

字段提取自 raw-source.md 各来源已按字段方向留底的内容，organize 做的是**归纳浓缩**，不是重新采集。

frontmatter 各字段来源：选题、来源类型、来源路径或链接取自 raw-source.md 的 frontmatter；整理时间由用户提供或交互确认，**不在 skill 内取系统时间**（与选题命名日期同规）。

## 2. 三种来源的整理侧重

### 2.1 book（感悟文档）

素材已是读者自己的结构化表达，整理最顺：

- **核心观点**：通常直接取自感悟文档「核心观点」章节 + 文首概括，浓缩成一句话。
- **关键论据素材**：取「论证脉络」章节，归纳成论据清单。
- **可引用金句**：取「我的理解」中读者的精炼表达——这是最有价值的素材，因为是读者自己的话，原创性天然成立。
- **受众**：感悟文档通常不含，标注「待补」，请用户在确认时填。
- **备注**：带入「待深入」章节内容，提醒 adapt 不要在这些点上强行展开；可附书名章号便于追溯。

### 2.2 url（链接）

素材是别人的内容，整理时严守原创性：

- **核心观点**：必须是**用户基于素材的再表达**，不是照搬原文主张。整理时把原文主张记入「关键论据素材」，核心观点留待用户确认时确立自己的角度。备注里写「外部链接素材，核心观点须用户再表达，勿照搬」。
- **关键论据素材**：归纳原文论据、数据、案例，标注来自原文。
- **可引用金句**：挑原文精炼句，**标注「来自原文，引用注意版权与转述」**。
- **受众**：从文章主题和语气推断。
- **备注**：来源平台、作者、发布时间、时效性、原创性提醒。

### 2.3 topic（话题）

信息最少，整理重点是帮用户聚焦：

- **核心观点**：话题来源常缺失。整理时把搜到的各方观点呈现给用户，**请用户在确认时确立自己的核心观点与差异化定位**。不要替用户编观点。备注里写「话题来源，核心观点待用户确立」。
- **关键论据素材**：归纳搜到的数据、案例、事实。
- **可引用金句**：搜到的精炼表述，标注来源。
- **受众**：从话题属性推断。
- **备注**：各方观点概览、竞品常见角度、差异化建议（别人没写透的缺口）。

## 3. 字段缺失处理

按 `../../methodology/references/pipeline.md` 第 5.1 节：

- **核心观点缺失** → 不进 adapt，回 organize 补。话题来源核心观点缺失属正常，引导用户确认时确立。
- **其余字段缺失** → 在 source.md 标注「待补」，organize 可继续，向用户提示该字段缺失可能影响成品质量。

不强填、不编造。「待补」是诚实标注，比编一个似是而非的内容强。

## 4. 话题来源确立观点的引导话术

话题来源整理完，展示 source.md 时，核心观点字段若为空或为「待用户确立」，用如下话术引导用户：

> 这是个话题素材，目前还没有明确的核心观点。我搜到了这些各方观点：
> - 观点A：<...>
> - 观点B：<...>
> 竞品多从 <角度> 切入，<角度> 这个缺口别人没写透。
> 你想确立什么核心观点？基于哪个差异化角度？确立后我再按你指定的平台适配。

把确立观点的主动权交给用户，organize 只提供素材和角度参考。

## 5. 多来源合并

用户一次整理多个来源（如多章感悟、多链接、感悟+链接混合）时：

- 逐个来源按上述规则提取，再合并进同一 source.md。
- 合并时去重、归并相近论据。
- 核心观点若多来源冲突，呈现冲突请用户在确认时定夺。
- 备注里列明各来源及占比，便于追溯。

## 6. 整理质量自检

整理完、展示给用户前，自检：

- [ ] 核心观点是一句话主轴，能统领全篇？（话题来源允许暂缺并引导）
- [ ] 关键论据素材都来自 raw-source.md，无编造？
- [ ] 可引用金句标注了来源与原创性提醒？
- [ ] 待补字段如实标注，未强填？
- [ ] 链接/话题来源的原创性提醒写进备注了？
- [ ] 感悟文档的「待深入」点带入了备注？

自检通过再展示给用户确认。
