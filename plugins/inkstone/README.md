# inkstone

中文社媒文案改造器。把阅读感悟、链接、话题等素材，按「前采集 → 中整理 → 后适配」三阶段，改造成适合在微信公众号、知乎、小红书发布的文案。前端按来源扩展、后端按平台扩展，结构对称。

## 1. 解决什么问题

你有一堆原始素材——bookworm 产出的阅读感悟、一个网页链接、一个想写的话题——但每个平台的调性天差地别：公众号要长文结构、知乎要论证深度、小红书要短平快和话题标签。本插件不替你凭空创作，而是把同一份素材**改造**成适配各平台调性的可发布文案，让你一份素材多平台分发。

## 2. 三阶段流水线

| 阶段 | Skill | 做什么 |
|------|-------|--------|
| ① 前采集 | `/inkstone:gather` | 把感悟文档 / 链接 / 话题采集成原始素材，决定文案归属 |
| ② 中整理 | `/inkstone:organize` | 把异构素材整理成统一中间格式（source.md） |
| ③ 后适配 | `/inkstone:adapt` | 按指定平台风格规范，把中间格式改造成平台文案 |

另有 `methodology` 总览 skill 自动激活，判断你处于哪阶段并引导到正确环节；缺前置阶段时自动串起全链路。

## 3. 设计原则

- **改造而非创作**：输入是已有素材，输出是适配调性的文案，不凭空生成
- **前后端对称扩展**：加来源只在 gather 内加 reference；加平台只在 adapt 内加 reference，互不干扰
- **统一中间格式**：所有来源先归一成 source.md，平台适配无差别消费，每个平台不用重复处理来源差异
- **口语化真人感**：默认自然口语化中文，避免 AI 腔
- **用户确认中间格式**：organize 整理完需用户确认后才进适配，把质量分水岭交给用户把控

## 4. 目录约定

阅读笔记（输入）与文案（输出）分目录存放，与 bookworm 共用同一套项目级约定：

```
<project>/
├── books/                        # bookworm 产出的阅读笔记（inkstone 的输入来源）
│   └── <书名>/
│       └── notes/
│           └── chXX.consolidate.md   # 感悟文档（gather 的主要素材）
└── copywriting/                  # inkstone 产出的平台文案
    ├── <书名>/                   # 来源于某本书
    │   └── <选题>/
    │       ├── source.md          # 统一中间格式（organize 产出）
    │       ├── wechat.md          # 公众号文案
    │       ├── zhihu.md           # 知乎文案
    │       └── xiaohongshu.md     # 小红书文案
    └── <其他>/                   # 来源于链接/话题等
        └── <选题>/
            ├── source.md
            └── ...平台文案
```

gather 不主动扫描感悟文档，而是由用户显式传素材路径，按路径前缀判断文案归属：路径在 `books/<书名>/` 下 → 落 `copywriting/<书名>/`，其余 → 落 `copywriting/<其他>/`。

## 5. 环境要求

- 无额外运行时依赖。感悟文档由 Claude 原生读取能力处理，链接抓取用内置 `WebFetch`，话题搜索用内置 `WebSearch`。
- 链接遇付费墙 / 反爬 / 登录墙时，插件会报错并提示你手动粘贴正文，不静默失败。

## 6. 快速上手

1. 把素材交给 Claude，运行 `/inkstone:gather <素材路径或链接或话题> [选题名]`
2. 确认 organize 整理出的中间格式（source.md）
3. 指定平台适配，如 `/inkstone:adapt 公众号`（或知乎、小红书）
4. 文案产出在 `copywriting/<书名或其他>/<选题>/<平台>.md`

也可以直接说「帮我把这个感悟改成知乎回答」「把这个链接变成小红书笔记」，methodology 会自动串起三阶段。
