# 章节识别与切分规则

disassemble 把整本书原文按章节切分存档。本文件定义章节标记的识别模式和切分算法。

## 1. 支持的输入格式

| 格式 | 归一化方式 | 依赖 |
|------|-----------|------|
| txt / md | Claude 直接读取 | 无 |
| PDF | Read 工具按页读取 | 无 |
| EPUB | 运行 `scripts/parse_epub.py` 转纯文本 | Python 标准库 |
| 其他 / 失败 | 提示用户先用阅读器导出成 txt/md | — |

归一化后统一得到纯文本，再进入章节切分。

## 2. 章节标记模式

识别以下章节标记（按优先级，先匹配到的优先）：

### 2.1 中文“第X章”

匹配正则（大小写、全角半角、空格容错）：

```
^\s*第\s*[一二三四五六七八九十百千零0-9]+\s*[章节回卷部篇]\s*[^\n]*$
```

样例：`第一章 导论`、`第 3 章 市场先生`、`第十二章`、`第二回`。

### 2.2 英文“Chapter X”

```
^\s*Chapter\s+[IVXLCDM0-9]+(\s|:|\.|—|–)?\s*[^\n]*$
```

样例：`Chapter 1`、`Chapter I: The Origin`、`CHAPTER 2 — Overview`。大小写不敏感。

### 2.3 Markdown 标题 / 独立行标题

- Markdown 一级标题：`^#\s+.+$`
- Markdown 二级标题（若无一级）：`^##\s+.+$`
- 独立行（无标点、短文本、单独成行且前后空行）：仅当 2.1/2.2 都未匹配时作为兜底，且需结合上下文判断，避免误把段落首行当标题。

## 3. 切分算法

1. **扫描全文**，逐行匹配章节标记模式。
2. **命中处作为章节边界**：从当前标记行到下一个标记行（或文末）之间为一个章节。
3. **标记行本身保留为章节标题**，写入切分后的 md 文件首行。
4. **章号推断**：从标记中提取数字（中文数字转阿拉伯，罗马数字转阿拉伯）。中文数字转换表见下。
5. **未匹配到任何标记**：报告“未识别到章节标记”，向用户确认是否整本作为单章，或让用户提供章节标记样例后重试。

## 4. 中文数字转阿拉伯

```
一→1 二→2 三→3 四→4 五→5 六→6 七→7 八→8 九→9 十→10
十一→11 十二→12 ... 二十→20 二十一→21 ...
百、千 按位权换算（如 一百二十三→123）
```

简单情况（1-99）直接映射；复杂情况（百、千）按位权计算。若遇无法转换的数字，使用章节序号（第 N 个标记即 ch N）兜底。

## 5. 章号兜底规则

若标记中提取不出数字，则按标记出现顺序编号：第 1 个标记 → ch01，第 2 个 → ch02，以此类推。文件名用两位补零：`ch01.md` ... `ch99.md`，超过 99 章用三位。

## 6. 切分产物格式

每个 `raw/chXX.md` 首行为章节标题（作为一级标题），其后为正文：

```markdown
# 第一章 导论

<章节正文，保留原段落结构>
```

## 7. 健壮性处理

- **章节标记密集误匹配**：若识别出的章节数异常多（如 >200）或异常少（如 1），向用户报告并请其确认。
- **前置内容**：第一个章节标记之前的内容（封面、目录、序言）单独存为 `raw/ch00-front.md`，不丢弃。
- **编码问题**：归一化输出统一 UTF-8。
- **空章节**：跳过正文为空的章节，不计入章号。
